`defineConfig` 是项目级配置入口。你从根目录的 `fasteval.config.ts` 默认导出它。

```ts
import { defineConfig } from "fasteval";
```

## 所有常用选项

<ParamField body="agents" type="AgentAdapter[]" default="[]">
  当前项目可用的自定义 agent adapters。每个 adapter 的 `name` 必须唯一。内置 coding agents 不需要注册。
</ParamField>

<ParamField body="defaultAgent" type="string">
  eval 未指定 `agent` 且 CLI 未传 `--agent` 时使用的 agent 名。
</ParamField>

<ParamField body="judge" type="{ model: string }">
  配置 `t.judge.*` 使用的默认 judge 模型。

  优先级从高到低：
  1. 单次 `t.judge.*` 调用的 `{ model }`
  2. 单个 `defineEval` 上的 `judge.model`
  3. `defineConfig` 中的 `judge.model`
</ParamField>

<ParamField body="reporters" type="Reporter[]" default="[Console()]">
  接收 eval 结果的 reporters。
</ParamField>

<ParamField body="maxConcurrency" type="number" default="4">
  eval 级别的最大并发。
</ParamField>

<ParamField body="timeoutMs" type="number" default="300000">
  单个 eval 的超时时间。
</ParamField>

<ParamField body="sandbox" type='"docker" | "vercel" | "auto"'>
  默认 sandbox 后端。
</ParamField>

<ParamField body="budget" type="number">
  整次运行的预算上限。
</ParamField>

<ParamField body="hooks" type="object">
  生命周期 hooks，用于准备环境、启动服务、清理资源或给 sandbox 注入依赖。
</ParamField>

## 示例

```ts
import { defineConfig } from "fasteval";
import { Console, JUnit } from "fasteval/reporters";
import supportApi from "./agents/support-api";

export default defineConfig({
  agents: [supportApi],
  defaultAgent: "support-api",
  judge: { model: "anthropic/claude-haiku-4-5" },
  reporters: [Console(), JUnit(".fasteval/junit.xml")],
  maxConcurrency: 4,
  timeoutMs: 300_000,
  sandbox: "auto",
});
```

## 配置放哪里

`fasteval.config.ts` 应放在项目根目录。CLI 会从当前工作目录向上查找配置。
