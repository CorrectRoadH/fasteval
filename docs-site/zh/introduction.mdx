fasteval 是一个轻量的 TypeScript 库，为 AI agents 提供结构化、可重复的评测。你不需要手动点几次 agent 再凭感觉判断好坏，而是写声明式 eval 文件描述“什么算好结果”。fasteval 负责运行、评分，并产出可读报告。

它可以用于 Claude Code 插件、面向用户的 HTTP agent 服务，也可以用于内部函数或 LLM wrapper。三类对象都通过同一个 `defineEval` API 接入。

  <Card title="HTTP 服务与部署后的 agent" icon="globe">

  </Card>
  <Card title="进程内函数" icon="function">
    直接在 eval 进程里调用你的 TypeScript 函数。把 eval 当成语义级单元测试放进 CI。
  </Card>

## 为什么叫 fast

**写得快。** 每个 eval 是一个文件，ID 自动从路径推导：`evals/weather/brooklyn.eval.ts` 变成 `weather/brooklyn`。断言写在一条线性的 `async test(t)` 里。数据集可以用一行 `.map` 扇出。

**跑得快。** runner 用有界并发并行执行，不会压垮被测 agent。基于 fingerprint 的缓存会跳过已经通过的 case。沙箱可以复用和预热。`earlyExit` 会在某次尝试成功后停止同一任务的剩余重试。

**看得快。** 控制台实时流式输出。每次运行都会生成事件流、完整 transcript、文件 diff 和断言结果，落在 `.fasteval/<timestamp>/` 目录里。统一 trace 让你能还原 agent 到底做了什么。

## 架构形状

```
   your evals/ directory            fasteval core               agent adapter
   ─────────────────────           ──────────────             ─────────────────
   weather.eval.ts   ──discover──>  Runner  ──send──>  Agent  ─── in-process
   sql.eval.ts                        │                        ─── remote HTTP
   fixtures/button/  ──fixture──>     │                        ─── sandbox ──> Docker
     PROMPT.md                        ▼
     EVAL.ts                       Scorers ──> Reporters ──> .fasteval/<run>/
```

- **fasteval core** 负责所有被测对象共通的部分：eval 发现、断言收集、评分、并发调度、缓存、报告和 artifact 落盘。
- **Agent adapter** 是 core 和被测系统之间的开放边界。fasteval 内置 Claude Code、Codex 和 bub 的适配器，你也可以写自己的 adapter。
- **Sandbox** 负责隔离运行 coding agent。默认使用 Docker，也支持第三方沙箱后端。

<Note>
  fasteval core 不硬编码某个具体 agent 的行为。新增 agent 类型应该通过 adapter 完成，而不是改 runner 或 scorers。
</Note>

## 两种接入模式

**Sandbox mode** 适合 Codex、Claude Code 这类必须在真实文件系统中改代码、跑命令的 coding agent。

**Direct mode** 适合 HTTP 服务和进程内函数，不需要 Docker，也不需要隔离工作区。
