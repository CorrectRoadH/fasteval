安装 fasteval 通常分三步：确认环境、安装依赖、运行 `npx fasteval init` 生成起始结构。

## 前置要求

**Node.js** — fasteval 需要支持 ESM 的现代 Node.js。优先使用项目 `.nvmrc` 或 `package.json` `engines` 指定的版本。

**Docker** — 只有 sandbox eval 需要 Docker，也就是评估 Claude Code、Codex、bub 这类需要隔离文件系统的 coding agent。进程内函数和 HTTP agent eval 不需要 Docker。

<Note>
  如果运行 sandbox eval 时 Docker 不可用，fasteval 会直接报错，不会静默切换到其他后端。
</Note>

## 安装

<CodeGroup>

```shell npm
npm install -D fasteval
```

```shell yarn
yarn add --dev fasteval
```

```shell pnpm
pnpm add -D fasteval
```

</CodeGroup>

## 初始化项目

```shell
npx fasteval init
```

`init` 会生成最小可运行结构：

```
your-project/
├─ fasteval.config.ts
└─ evals/
   ├─ hello.eval.ts
   └─ fixtures/
      └─ button/
         ├─ PROMPT.md
         ├─ EVAL.ts
         └─ package.json
```

示例文件用于展示 eval 形状。理解之后可以直接替换或删除。

<Tip>
  Eval ID 自动从路径推导。`evals/weather/brooklyn.eval.ts` 的 ID 是 `weather/brooklyn`，不需要手写 ID。
</Tip>

## 配置 fasteval

打开生成的 `fasteval.config.ts`：

```ts
import { defineConfig } from "fasteval";
import { Console, JUnit } from "fasteval/reporters";

export default defineConfig({
  judge: { model: "anthropic/claude-haiku-4-5" },
  reporters: [Console(), JUnit(".fasteval/junit.xml")],
  maxConcurrency: 4,
  timeoutMs: 300_000,
  sandbox: "auto",
});
```

常用配置：

| 选项 | 作用 |
|---|---|
| `agents` | 注册自定义 agent adapters |
| `defaultAgent` | CLI 未传 `--agent` 时使用哪个 agent |
| `judge.model` | `t.judge.*` 使用的评判模型 |
| `reporters` | 控制台、JUnit 等 reporter |
| `maxConcurrency` | 同时运行多少个 eval |
| `sandbox` | `docker`、`vercel` 或 `auto` |

## 环境变量

不同 agent 需要不同凭据：

```shell
export ANTHROPIC_API_KEY=sk-ant-...
export OPENAI_API_KEY=sk-...
export VERCEL_TOKEN=...
```

<Warning>
  不要把 API key 写进 `fasteval.config.ts`。在本地用 shell 环境变量，在 CI 里用 secrets。
</Warning>

## 验证安装

```shell
npx fasteval --help
npx fasteval list
npx fasteval --dry
```

这些命令能正常执行后，就可以进入 [快速开始](/zh/quickstart)。
